## 二、预算规则

### 数据结构

每条预算：

```js
{
  id: "budget_xxx",
  category: "Food",       // 主分类标识
  subCategory: "午餐",    // 子分类名，null 表示主分类预算
  period: "monthly",      // "monthly" | "yearly"
  amount: 3000,           // 预算金额（元）
  createdAt: "2026-07-17"
}
```

### 规则

1. 一个分类最多一条预算，选周期：每月 / 每年
2. 子分类可选，有子分类则预算下钻到子分类
3. 概览页展示时，年预算自动除以 12 换算为月均
4. 分类和子分类用户可自由新增/编辑/删除
5. 收入分类不需要预算

## 三、页面设计

### 3.1 概览页 — 预算区域

位置：放在净资产/储蓄率区域下方

组件：
- 顶部：本月总预算环形摘要（总计/已用/剩余，百分比）
- 下面：各分类进度条列表
  - 每行：分类图标 + 名称 + 进度条 + 金额
  - 超支进度条变红色
  - 子分类不展示在概览（概览只看主分类汇总）
- 底部：「管理预算 →」链接跳转个人中心

### 3.2 个人中心页面

```
┌──────────────────┐
│ 头像  昵称       │
│ email            │
├──────────────────┤
│ 📊 预算管理   >  │
│ 📤 数据导出   >  │
│ 🏷️ 类目管理   >  │
│ ⚙️ 账号设置   >  │
└──────────────────┘
```

### 3.3 预算管理页面

列表页：
- 每个分类一行，显示分类图标 + 名称 + 周期 + 金额
- 已设置：显示「月 ¥3,000」或「年 ¥20,000」
- 未设置：显示「未设置」
- 有子分类：下方小字显示子分类概要
- 点击进入编辑页

编辑页（按分类）：
- 表单：周期选择（每月/每年）+ 金额输入
- 子分类列表（如果存在）：每行显示名称 + 月预算金额 + 删除按钮
- 「+ 添加子分类」按钮：弹出输入框（子分类名 + 月预算金额）
- 点击删除子分类前确认

### 3.4 类目管理页面

- 展示所有分类（支出 + 收入）
- 每个分类：图标 + 名称 + 编辑/删除按钮
- 内置分类（8 支出 + 4 收入）可编辑不可删除
- 用户自定义分类可编辑可删除
- 「+ 新增分类」按钮：选择支出/收入类型 → 选图标 → 填名称

### 3.5 数据导出页面

- 从 transactions.js 的 _exportCSV 迁移
- 导出全量 CSV（原有功能）
- 可选：按时间范围导出

## 四、技术实现

### 存储

预算数据存储在 localStorage（未登录）/ Firestore（已登录），与现有 store 体系一致。

### 预算计算

- 统计时根据预算 period 换算：
  - monthly：直接匹配当月交易
  - yearly：年预算 ÷ 12 = 月均预算，对比当月实际支出
- 汇总所有分类预算 = 总预算

### 超支检测

- 进度 = 当月实际支出 / 月预算
- > 100%：红色，> 80%：橙色，≤ 80%：默认色
